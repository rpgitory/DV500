1 Expand NAT in NAT gateways 
Ans. NAT stands for Network Address Translation. In the context of NAT Gateways in AWS, NAT refers to the process of translating IP addresses between private and public networks. NAT Gateways facilitate outbound internet connectivity for instances in private subnets of a Virtual Private Cloud (VPC) by performing network address translation.
	
2 How to create NAT gateway?
Ans. go to left panel dash board then click to NAT gateways and click to create a NAT gateways then select public server subnet then click on auto assign elastic ip (EIP). after that add NAT in route tables
